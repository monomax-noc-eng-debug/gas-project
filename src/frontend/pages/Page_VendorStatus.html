รายละเอียดแยกระบบ

ข้อมูลการทำงานของระบบ Monitor

แหล่งที่มาของข้อมูล (Data Source)

Official API

ดึงข้อมูลจาก Status Page ของผู้ให้บริการโดยตรง (JSON API) เพื่อความแม่นยำสูงสุด (เช่น Mux, Akamai)

HTTP Health Check

ตรวจสอบผ่านการส่งคำขอ HTTP เพื่อเช็คการตอบสนองและวัดความเร็ว (Latency) แบบ Real-time

เกณฑ์การแบ่งสีสถานะ (Status Logic)

Operational (ปกติ)
API แจ้งว่าปกติ หรือ HTTP ตอบกลับ 200 OK ภายใน 3 วินาที
Degraded (เฝ้าระวัง)
API แจ้งปัญหาเล็กน้อย หรือ HTTP ตอบช้าเกิน 3 วินาที (Latency สูง)
Critical (ล่ม/ขัดข้อง)
API แจ้งปัญหาหลัก หรือไม่สามารถเชื่อมต่อได้/พบ Error รหัส 5xx หรือ 4xx

ระบบจัดการแคช (Caching System)

ข้อมูลจะถูกจดจำไว้เป็นเวลา 2 นาที เพื่อประหยัดทรัพยากรและให้หน้าจอโหลดได้รวดเร็วเมื่อสลับหน้าไปมา ระบบจะทำการ Auto Refresh ทุกๆ 5 นาที ขณะเปิดหน้าจอนี้ทิ้งไว้เพื่อให้ข้อมูลอัปเดตตลอดเวลา
